FAQ

Common questions

How long does integration take?+

For most teams, full integration takes 10–30 minutes. Install the SDK, register your agents, and add one line per action. We have step-by-step guides for LangChain, CrewAI, AutoGen, and plain Python.

Does it add latency to my agents?+

No measurable impact. The SDK call responds in under 2ms. Your agent proceeds immediately while scoring happens in parallel. We've tested it on production agent networks with no observable slowdown.

What happens when an agent is suspended?+

The agent's status is set to "suspended" and all subsequent behavior logs return a block signal. Your code checks result.is_safe before proceeding. You receive a webhook alert and the full incident is preserved in the audit trail.

How does the ML model learn normal behavior?+

The model trains on each agent's individual history using Isolation Forest with 15 features. After 10 logs, scoring begins. After 100 logs, the baseline is mature and detection accuracy is high. Each agent has its own independent model.

Is my agent's behavior data kept private?+

Yes. All data is isolated per organization and encrypted at rest. We never share data between organizations and never use your data to train shared models. Enterprise customers can request on-premise deployment for maximum data sovereignty.

DOCUMENTATION

Developer Documentation

Everything you need to integrate TrustAgent. Go from zero to production-secure in under 30 minutes.

Quick Start

pip install trustagent

from trustagent import TrustAgent

ta = TrustAgent(api_key="your_api_key")
agent = ta.register("MyAgent", description="My first agent")
result = agent.log("process_request", {"user": "alice"})

print(result.trust_score)    # 100.0
print(result.is_anomalous)   # False
print(result.status)         # active

Authentication

All API calls require your organization's API key as the x-api-key header. Get your key from the dashboard after signing up. Store it as an environment variable — never hardcode it.

SDK Reference

TrustAgent(api_key)

Initialize the client with your API key. The client is thread-safe and can be shared across your application.

ta.register(name, description)

Register a new agent with your organization. Returns an Agent object with a unique ID and Ed25519 keypair. Each agent is cryptographically unique.

agent.log(action, payload)

Log a behavior event for an agent. Returns a result with trust_score, is_anomalous, status, is_safe, and reason. Call this before every significant action your agent takes.

agent.audit_trail()

Retrieve the full behavior history for the agent. Returns a list of all logged events with timestamps, scores, and reasons.

agent.refresh()

Fetch the latest agent state from the API including current trust score and status.

REST API

The full REST API is live at https://trustagent-production.up.railway.app. Interactive Swagger docs available at /docs.

Framework Guides

  • LangChain — wrap your tools with agent.log() calls
  • CrewAI — add TrustAgent to your crew task callbacks
  • AutoGen — integrate via message interceptor hooks
  • OpenAI Agents — add to your function call handlers

SECURITY

Security at TrustAgent

We hold ourselves to the highest security standards. Here's exactly how we protect your data and your agents.

Cryptography

All agent identities use Ed25519 elliptic curve cryptography. Keys are generated server-side and private keys are returned once — we do not store your private keys after generation.

Transport Security

All API traffic is encrypted with TLS 1.3. We enforce HSTS and do not support any protocol below TLS 1.2.

Infrastructure

TrustAgent runs on Railway's managed infrastructure with automatic SSL, DDoS protection, and private networking between services. PostgreSQL is encrypted at rest.

Rate Limiting

All endpoints are rate limited to prevent abuse. Authentication endpoints have stricter limits to prevent brute force attacks.

Responsible Disclosure

Found a security vulnerability? Please email [email protected]. We will respond within 24 hours and credit researchers who report valid issues.
